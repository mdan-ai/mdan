# MDAN Spec v1

MDAN Spec v1 is the current public, versioned entry for agent-readable MDAN interaction over HTTP.

If you want the complete specification body, use:

- [MDAN Spec](./spec.md)

## What `v1` Is For

`v1` is the public profile entry that tells a client, agent, or implementer how to recognize and correctly consume MDAN over HTTP.

Its primary job is to answer:

- what this Markdown response means
- how a client should treat it
- which media type and profile identify it
- what kinds of interaction structure may appear inside it

## Media Type

MDAN Spec v1 is declared as:

```http
Content-Type: text/markdown; profile="https://mdan.ai/spec/v1"
```

The base media type remains `text/markdown`.

The `profile` parameter tells the client that this Markdown carries MDAN spec semantics.

## What This Means

A response with this profile may contain:

- one canonical page or fragment written in Markdown
- one executable `mdan` fenced block
- `BLOCK` declarations
- `INPUT` declarations
- `GET` and `POST` operation declarations
- block anchor mapping through `<!-- mdan:block ... -->`
- follow-up interaction context for the next step

This is not just reading content. It is content that may also be interactive and executable within the MDAN model.

## Client Expectations

A client receiving this profile should:

- treat the Markdown response as canonical MDAN content
- preserve executable `mdan` blocks and block anchors
- understand that the response may describe next-step interaction, not just current state
- prefer this Markdown representation over derived HTML when both are available

## HTTP Scope

MDAN v1 currently covers:

- full page Markdown responses
- fragment Markdown responses
- HTTP `GET` and `POST` interaction
- Markdown and HTML dual representation of the same app

MDAN v1 does not define:

- transport beyond HTTP
- browser rendering requirements
- authentication strategy
- non-HTTP execution environments

## Relationship To HTML

HTML is the browser-facing projection of the same app.

For HTML responses, a host should help agents discover the canonical Markdown source through:

- `rel="alternate" type="text/markdown"`
- optional `rel="llms-txt"` when the site explicitly provides a real `llms.txt`
- optional equivalent `Link` headers

The profile itself belongs on the Markdown response, not on the HTML response.

## Complete Spec

For the full grammar, validation constraints, HTTP binding, execution semantics, session semantics, and host conformance requirements, use:

## Versioning

`v1` is the first public spec profile for MDAN.

Future spec revisions should use a new stable profile URL instead of changing the meaning of `v1`.

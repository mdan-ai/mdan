# 0.6.0 Release Notes

## The Syntax Transition Release

`@mdanai/sdk@0.6.0` is the major syntax transition release for the current MDAN line.

This release makes the current MDAN syntax the default across the main SDK surface and completes the transition away from the temporary legacy or `v2` naming used during the migration period.

## Breaking Change

- the new MDAN syntax is now the default syntax across the main SDK surface
- legacy syntax support has been removed from the mainline SDK surface
- existing apps that still rely on the legacy syntax need to migrate before upgrading to `0.6.0`

## What Changed

- core syntax modules and exports now use their canonical names without `v2` suffixes
- public examples, starter templates, docs, and mainline tests now use the current syntax by default
- direct-write request semantics are now documented consistently across `text/markdown`, `application/x-www-form-urlencoded`, and `multipart/form-data`

## Starter Compatibility

`create-mdan@0.6.0` now generates apps against:

- `@mdanai/sdk: ^0.6.0`

That keeps newly scaffolded apps aligned with the SDK line that uses the current syntax by default.

## Scope

- this is a coordinated `0.6.0` release for both `@mdanai/sdk` and `create-mdan`
- the release focuses on syntax-line consolidation, package surface cleanup, updated starter defaults, and aligning the entire public surface on one syntax
- Node and Bun remain the officially supported runtime targets through the same shared server runtime and runtime-specific host adapters

## Upgrade Note

If you already have an MDAN app on the legacy syntax line, read this release as an upgrade boundary rather than a routine dependency bump.

Before upgrading:

- update page sources to the current MDAN syntax
- update any code that still imports transitional `syntax-v2` names
- verify server flows, starter-derived pages, and direct-write request handling against the current syntax behavior

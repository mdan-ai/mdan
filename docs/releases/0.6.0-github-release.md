# GitHub Release Draft: MDAN SDK 0.6.0

## Suggested Title

`0.6.0: new MDAN syntax becomes the default`

## Suggested Body

`0.6.0` is the syntax transition release for the current MDAN line.

This release makes the current MDAN syntax the default across the main SDK surface, removes legacy syntax support from the mainline SDK, and aligns the starter, examples, docs, and tests around one syntax.

### Breaking change

- the new MDAN syntax is now the default
- legacy syntax support has been removed from the mainline SDK surface
- existing apps that still rely on the legacy syntax need to migrate before upgrading to `0.6.0`

### What changed

- core syntax modules and exports now use their canonical names without `v2` suffixes
- `create-mdan@0.6.0` now generates apps against `@mdanai/sdk: ^0.6.0`
- public examples, starter templates, docs, and mainline tests now use the current syntax by default
- direct-write request semantics are now documented consistently across `text/markdown`, `application/x-www-form-urlencoded`, and `multipart/form-data`

### Upgrade note

If you are already running an MDAN app on the legacy syntax line, treat `0.6.0` as an upgrade boundary rather than a routine dependency bump.

Before upgrading:

- update page sources to the current MDAN syntax
- update any code that still imports transitional `syntax-v2` names
- re-verify server flows and starter-derived pages under the current syntax behavior

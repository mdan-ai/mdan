# 0.4.0 Release Notes

## Official Node + Bun Support

MDAN `0.4.0` is the first release that officially supports both Node and Bun as runtime targets.

The SDK now keeps its shared server modeling APIs on `@mdanai/sdk/server`, while runtime-specific host adapters live on:

- `@mdanai/sdk/server/node`
- `@mdanai/sdk/server/bun`

That gives Node and Bun first-class support without splitting the core API surface or maintaining separate products.

## Starter Updates

`create-mdan` now scaffolds runtime-specific starters:

- `npm create mdan@latest` and `npx create-mdan` default to the Node starter
- `bunx create-mdan` defaults to the Bun starter
- `--runtime node` and `--runtime bun` are supported explicitly

The starter structure is still centered around the same shared `app/` model, so moving between Node and Bun does not require relearning the application layout.

## Verification and Packaging

This release also adds:

- Node and Bun smoke coverage for release flows
- Explicit Bun host tests in the SDK test suite
- Root-level license management, with package-local `LICENSE` files synced during packing

## Upgrade Notes

If you host MDAN apps directly, prefer the explicit runtime adapter imports:

```ts
import { createHostedApp } from "@mdanai/sdk/server";
import { createHost } from "@mdanai/sdk/server/node";
```

or:

```ts
import { createHostedApp } from "@mdanai/sdk/server";
import { createHost } from "@mdanai/sdk/server/bun";
```

If you use `create-mdan`, you can now choose the runtime at scaffold time instead of treating Bun as a package-manager-only workflow.

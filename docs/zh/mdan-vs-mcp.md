# MDAN 与 MCP

MDAN 和 MCP 都和 AI 应用有关，但它们解决的不是同一层问题。

## MCP 擅长什么

MCP 是一种把工具、资源和结构化能力暴露给 AI 系统的协议。

它适合用在这些场景：

- 给 AI 客户端暴露工具能力
- 统一 Assistant 连接外部能力的方式
- 建模 integrations、resources 和 tool calling

## MDAN 擅长什么

MDAN 是一种共享表示法，用来表达可交互页面，并让同一页面从同一份源在不同界面中都能同时对人类和 Agent 保持可读、可操作。

它适合用在这些场景：

- 同一个应用要同时服务浏览器和 Agent
- 你想把内容和后续交互放在同一个可读页面模型里
- 服务端返回的 Markdown 片段本身就是下一步交互上下文
- 你想让同一个应用给 Agent 协商 Markdown，给浏览器协商 HTML

## 它们解决的是不同层

MCP 解决的是把能力暴露给 AI 客户端的问题。

MDAN 解决的是如何表达一个既能给人也能给 Agent 使用的交互式应用表面。

可以把它简单理解成：

- MCP 更像能力与工具协议
- MDAN 更像面向交互页面与持续交互的表示法

## 什么时候一起用

当你同时遇到下面这些需求时，可以一起用：

- Agent 通过 MCP 发现或调用能力
- 但真正面向用户的交互流程，仍然应该落在同一个浏览器与 Agent 共用的应用里
- 而且你希望这个应用本身保持 Markdown 可读、可继续执行

在这种结构里，MCP 可以帮助 AI 系统接入能力，MDAN 可以定义人和 Agent 继续工作的那个页面型应用表面。

## 什么时候 MDAN 更合适

优先选 MDAN，当你的核心问题是：

- 想做一个同时给人和 Agent 用的应用
- 想把页面内容和 action 放在一起定义
- 想把多步交互建模成可读的页面更新

## 什么时候 MCP 更合适

优先选 MCP，当你的核心问题是：

- 想把工具或资源暴露给外部 AI 系统
- 想标准化能力接入边界
- 想连接很多后端能力，但还不需要定义用户可见的应用模型

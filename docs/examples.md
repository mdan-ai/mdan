# Examples

The current repository keeps a small set of examples that exercise the intended
artifact-first runtime model.

## Starter

Route: [Starter Example](/examples/starter)

Use it when you want the smallest app that still demonstrates:

- Markdown as the canonical artifact
- a declared action contract
- browser HTML projection from the same underlying state

## Docs Starter

Route: [Docs Starter Example](/examples/docs-starter)

Use it when you want a docs-shaped app that loads multiple Markdown pages and
exposes a simple read action.

## Auth Guestbook

Route: [Auth Guestbook Example](/examples/auth-guestbook)

Use it when you want the canonical auth flow in this repository:

- login
- registration
- authenticated session state
- guestbook posting flow

## Weather Markdown Demo

Route: [Weather Markdown Demo](/examples/weather-markdown)

Use it when you want a Markdown-first service example that returns deliverable
artifacts for both browsers and agents.

## Which One To Read First

- smallest runtime path: Starter
- docs-shaped app: Docs Starter
- auth and session behavior: Auth Guestbook
- agent-facing service output: Weather Markdown

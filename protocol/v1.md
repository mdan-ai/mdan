# MDAN Protocol v1

MDAN Protocol v1 defines how `text/markdown` documents carry interactive page semantics for MDAN apps.

When a client receives this profile, it should understand that the response is not plain reading-only Markdown. It is canonical MDAN page or fragment content that may contain executable interaction structure.

## Media Type

MDAN Protocol v1 is declared as:

```http
Content-Type: text/markdown; profile="https://mdan.ai/protocol/v1"
```

The base media type remains `text/markdown`.

The `profile` parameter tells the client that this Markdown carries MDAN protocol semantics.

## What This Means

A response with this profile may contain:

- one canonical page or fragment written in Markdown
- one executable `mdan` fenced block
- `BLOCK` declarations
- `INPUT` declarations
- `GET` and `POST` operation declarations
- block anchor mapping through `<!-- mdan:block ... -->`
- follow-up interaction context for the next step

In other words, this is not just content to read. It is content that may also be interactive and executable within the MDAN model.

## Client Expectations

A client receiving this profile should:

- treat the Markdown response as canonical MDAN content
- preserve executable `mdan` blocks and block anchors
- understand that the response may describe next-step interaction, not just current state
- prefer this Markdown representation over derived HTML when both are available

## Scope

Protocol v1 covers:

- full page Markdown responses
- fragment Markdown responses
- HTTP `GET` and `POST` interaction
- Markdown and HTML dual representation of the same app

Protocol v1 does not define:

- transport beyond HTTP
- browser rendering requirements
- authentication strategy
- non-HTTP execution environments

## Relationship To HTML

HTML is the browser-facing projection of the same app.

For HTML responses, a host should help agents discover the canonical Markdown source through:

- `rel="alternate" type="text/markdown"`
- optional `rel="llms-txt"` when the site explicitly provides a real `llms.txt`
- optional equivalent `Link` headers

The profile itself belongs on the Markdown response, not on the HTML response.

## Versioning

`v1` is the first public protocol profile for MDAN.

Future protocol revisions should use a new stable profile URL instead of changing the meaning of `v1`.

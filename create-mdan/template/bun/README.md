# __PROJECT_NAME__

Bun MDAN starter generated by `create-mdan`.

The app serves markdown surfaces and includes a thin frontend entry for browser UI.
Action submissions use JSON request bodies and return Markdown responses.
The generated app keeps `app/index.md` and `app/index.action.json` side by side,
and `app/server.mjs` uses the app API: `createApp`, `page`, `route`,
`action`, and `page.bind(...)`.

```bash
bun install
bun start
```

Open `http://127.0.0.1:4321/`.

Try this next:

- edit `app/index.md` and refresh the browser
- edit `app/index.action.json` to change the explicit action contract
- run `curl -H 'Accept: text/markdown' http://127.0.0.1:4321/` to inspect the canonical Markdown response
- if port `4321` is busy, start with `PORT=4322 bun start`
